# Асинхронная работа с внешней компонентой

## Общие правила

- Для Google Chrome и Mozilla Firefox допускается только асинхронное подключение и использование внешних компонент.
- В других браузерах синхронный или асинхронный режим определяется свойством конфигурации "Режим использования синхронных вызовов расширений платформы и внешних компонент".
- Переработка внешней компоненты не требуется, но нужна ее перекомпиляция для поддержки асинхронной работы.

## Через обратный вызов

- Подключение выполняется через `НачатьПодключениеВнешнейКомпоненты()`.
- Доступ к свойствам выполняется через `НачатьУстановку<ИмяСвойства>()` и `НачатьПолучение<ИмяСвойства>()`.
- Вызов методов выполняется через `НачатьВызов<ИмяМетода>()`.
- Методы `Начать...()` добавляются платформой автоматически.
- В `НачатьВызов...()` первым параметром передается описание оповещения, остальные параметры соответствуют оригинальному методу.

## Через обещания (8.3.18+)

- Установка компоненты выполняется через `УстановитьВнешнююКомпонентуАсинх()`.
- Подключение компоненты выполняется через `ПодключитьВнешнююКомпонентуАсинх()`.
- Объект компоненты создается через `СоздатьОбъектВнешнейКомпонентыАсинх()`, имя типа как для `Новый()`.
- Доступ к свойствам выполняется через `Получить<ИмяСвойства>Асинх()` и `Установить<ИмяСвойства>Асинх()`.
- Вызов методов выполняется через `<ИмяМетода>Асинх()`.
- Асинхронные методы добавляются платформой автоматически.
- Результат ожидания `<ИмяМетода>Асинх()` имеет тип `РезультатАсинхВызоваВнешнейКомпоненты`.
- Значение результата находится в `РезультатАсинхВызоваВнешнейКомпоненты.Значение`.
- Параметры (включая измененные) находятся в `РезультатАсинхВызоваВнешнейКомпоненты.Параметры`.
