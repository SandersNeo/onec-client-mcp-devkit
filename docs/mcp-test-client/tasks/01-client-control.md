# Задача 01: Управление тест‑клиентом (запуск/остановка)

Статус: `todo`

## Цель
Сделать управляемый запуск локального тест‑клиента 1С и подключение к нему через `ТестируемоеПриложение`, а также мягкую остановку/отключение.

## Область
Входит:
- Команда запуска клиента тестирования (локально).
- Подключение к клиенту тестирования с ретраями.
- Отключение и мягкая остановка клиента.
- Сохранение состояния в `Мсп_СостояниеКлиент` одной структурой.

Не входит:
- Работа с формами и элементами UI.
- Ресурсы UI.
- Уведомления.

## Входные данные и допущения
- Запуск через ключ `/TESTCLIENT`.
- Дополнительный параметр запуска: `/TESTCLIENT -TPort<Номер TCP-порта>`.
- Порт взаимодействия по умолчанию: `1538`.
- Параметры запуска передаются строкой командной строки.
- Источник базовой строки подключения — `СтрокаСоединенияИнформационнойБазы()`.
- Процесс запуска может быть длительным, требуется цикл повторных подключений.

## Интерфейсы

### Tools
- `test_client_start`
- `test_client_stop`

### Состояние
- Ключ `ПамятьТестКлиент` в `Мсп_СостояниеКлиент`.
- Структура состояния (минимум): `КомандаЗапуска`, `ТестКлиент`, `Подключен`, `ПоследняяОшибка`.

## Алгоритм запуска
1. Сформировать командную строку.
   - Включить `/TESTCLIENT` и, при задании порта, `-TPort<порт>`.
2. Выполнить запуск процесса 1С с ключом `/TESTCLIENT`.
3. Создать объект `ТестируемоеПриложение`.
4. Выполнить `УстановитьСоединение()` в цикле до 60 попыток с обработкой исключений.
5. При успехе сохранить состояние и вернуть результат.
6. При неудаче вернуть последнюю ошибку.

## Алгоритм остановки
1. Если есть объект `ТестируемоеПриложение`, выполнить `РазорватьСоединение()`.
2. Попытаться закрыть главное окно (через `ПолучитьАктивноеОкно().Закрыть()` или команду выхода).
3. Обновить состояние и вернуть результат.

## Правила обработки ошибок
- Для неудачи подключения возвращать последнюю ошибку без лишних сообщений.
- Для `test_client_stop` — возвращать предупреждение, если процесс не удалось корректно закрыть.

## Проверки
- Проверка, что команда запуска содержит `/TESTCLIENT`.
- Проверка корректности `TPort` (число, положительное).
- Проверка валидности параметров.
- Логирование ключевых шагов (запуск, подключение, остановка).

## Критерии готовности
- `test_client_start` запускает процесс и подключается к тест‑клиенту.
- `test_client_stop` разрывает соединение и пытается закрыть приложение.
- Состояние обновляется одной структурой.
- Есть минимальные лог‑сообщения по ключевым событиям.

## Риски и неизвестные
- Нет прямого управления PID процесса из BSL.
- Поведение `ЗапуститьСистему` и `ЗапуститьПриложение` может отличаться в клиентском контуре.
- Политика паузы между ретраями требует подтверждения.
