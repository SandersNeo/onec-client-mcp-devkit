# Задача 02: Ресурс чтения активного окна

Статус: `todo`

## Цель
Сделать ресурс `ui://active/window`, возвращающий JSON‑снимок активного окна с ограничением глубины.

## Область
Входит:
- Поддержка `resources/read` для `ui://active/window`.
- Получение активного окна через `ТестируемоеПриложение.ПолучитьАктивноеОкно()`.
- Построение снимка UI в формате `docs/mcp-test-client/contracts.md`.
- Ограничение глубины снимка параметром `depth` (через query‑строку URI).

Не входит:
- Ресурсы `ui://active/form`, `ui://forms/*`, `ui://elements/*`.
- Уведомления `notifications/ui/changed`.
- Инструменты взаимодействия с формами.

## Входные данные и допущения
- Тест‑клиент запущен и подключен (задача 01).
- Доступ к `ТестируемоеПриложение` через состояние `Мсп_СостояниеКлиент`.
- Если активное окно отсутствует, возвращается доменная ошибка `UI_NOT_FOUND`.

## Подзадачи
1. Описать ресурс в `resources/list` (uri, mimeType, краткое описание).
2. Реализовать `resources/read` для `ui://active/window`.
3. Реализовать сбор снимка:
   - получить активное окно;
   - сформировать дерево элементов с ограничением `depth`;
   - нормализовать поля (`type`, `name`, `title`, `visible`, `enabled`, `readOnly`, `children`, `path`).
4. Привести ответ к контракту MCP (`contents[]`, `mimeType=application/json`).
5. При необходимости обновить `docs/mcp-test-client/test-client-api.md`.

## Формат ответа
- `contents` — массив из одного элемента.
- `text` — JSON‑снимок активного окна.

## Обработка ошибок
- MCP `-32602` при неверном `depth`.
- MCP `-32603` при ошибке получения окна.
- `UI_NOT_FOUND` при отсутствии активного окна.

## Проверки
- Активное окно есть → возвращается корректный снимок.
- `depth=0|1|2` ограничивает вложенность.
- Нет активного окна → доменная ошибка `UI_NOT_FOUND`.

## Критерии готовности
- `resources/read ui://active/window` возвращает снимок по контракту.
- Параметр `depth` поддержан и имеет дефолт.
- Ошибки соответствуют контракту.

## Риски и неизвестные
- Отличия в типах окон/элементов между клиентами.
- Неочевидность картирования типов элементов в `type`.
