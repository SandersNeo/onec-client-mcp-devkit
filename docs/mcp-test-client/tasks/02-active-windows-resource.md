# Задача 02: Ресурсы окон

Статус: `in_progress`

## Цель
Перевести чтение окон на URI `window://active` и `window://{path}`, убрать `ui://` и сделать снимок окна точкой discovery форм.

## Область
Входит:
- Поддержка `resources/read` для `window://active`.
- Поддержка ресурса-шаблона `window://{path}`.
- Получение активного окна через `ТестируемоеПриложение.ПолучитьАктивноеОкно()`.
- Разрешение конкретного окна по `НавигационнаяСсылка`.
- Обновление `СформироватьСнимокОкна()` так, чтобы он возвращал `uri` окна и `forms[]` со списком форм окна.
- Удаление поддержки `ui://active/window` и `ui://windows` из публичного контракта.

Не входит:
- Отдельный ресурс списка окон.
- Ресурс формы `form://{name}`.
- Инструменты `find`, `activate`, `close`, `open_form`.

## Входные данные и допущения
- Тест‑клиент запущен и подключен (задача 01).
- Поддержка resource template в MCP-компоненте считается доступной к моменту реализации.
- `window://{path}` строится из `НавигационнаяСсылка` окна с percent-encoding path-сегмента.
- `window://active` используется как alias только для входа в чтение ресурса; в самих снимках возвращается конкретный `window://{path}`.

## Подзадачи
1. Обновить контракт ресурса окна в документации и общем плане.
2. Добавить описание ресурса `window://active` и шаблона `window://{path}`.
3. Реализовать разрешение окна по активному окну и по `path`.
4. Обновить `СформироватьСнимокОкна()`:
   - вернуть `uri`;
   - оставить поля окна (`type`, `title`, `isMain`, `homePage`);
   - добавить `forms[]` c элементами `{title, uri}`.
5. Удалить из контракта и регистрации отдельный ресурс списка окон.

## Формат ответа
- `contents` — массив из одного элемента.
- `text` — JSON‑снимок окна.

## Обработка ошибок
- `UI_NOT_FOUND` при отсутствии активного окна.
- `UI_NOT_FOUND` при отсутствии окна по `window://{path}`.
- `-32602` при некорректном URI окна.
- `-32603` при ошибке чтения окна.

## Проверки
- `window://active` возвращает снимок активного окна.
- `window://{path}` возвращает снимок конкретного окна.
- Снимок окна содержит конкретный `uri` и `forms[]`.
- `ui://` URI больше не документируются и не публикуются.

## Критерии готовности
- `resources/read window://active` возвращает снимок по новому контракту.
- `resources/read window://{path}` возвращает тот же формат снимка.
- В снимке окна присутствует список форм окна с `title` и `uri`.
- Отдельный ресурс списка окон не используется.

## Текущий прогресс
- Обновлён рабочий план задачи под новую схему URI.
- Этап 2 в общем плане переведён в `in_progress`.
- Зафиксировано, что discovery окон дальше будет идти через `find`, а не через отдельный list-resource.

## Риски и неизвестные
- Поддержка resource template зависит от доработки MCP-компоненты.
- Не все окна могут иметь пригодную для адресации `НавигационнаяСсылка`.
