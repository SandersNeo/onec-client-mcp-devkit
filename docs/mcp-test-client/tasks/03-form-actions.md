# Задача 03: Инструменты UI

Статус: `todo`

## Цель
Добавить URI-ориентированные инструменты `find`, `activate`, `close` и `open_form` для работы с окнами, формами и control.

## Область
Входит:
- Инструмент `find` как аналог `FindObjects`.
- Инструмент `activate` по URI объекта.
- Инструмент `close` для окна и формы.
- Инструмент `open_form({target})` для открытия формы через `ВыполнитьКоманду` активного окна.

Не входит:
- Чтение формы через `form://{name}`.
- Ввод текста, клики и прочая эмуляция пользователя.
- Политика подтверждений для опасных действий.

## Входные данные и допущения
- URI окон имеют вид `window://active` и `window://{path}`.
- URI форм имеют вид `form://{name}`.
- URI прочих элементов имеют вид `control://{form}/{name}`.
- `open_form` принимает объект аргументов с единственным строковым полем `target`.
- Значение `target` на этом этапе трактуется как навигационная ссылка.

## Подзадачи
1. Описать входные и выходные контракты инструментов в API-документации.
2. Реализовать `find(root_uri?, type?, title?, name?, timeout_sec?)`.
3. Реализовать `activate(uri)` для окна, формы и control.
4. Реализовать `close(uri)`:
   - `window://...` закрывает окно;
   - `form://...` закрывает родительское окно формы;
   - `control://...` отклоняется как неподдерживаемый URI.
5. Реализовать `open_form({target})` через `ВыполнитьКоманду()` активного окна.

## Формат результата
- `find` возвращает массив найденных объектов с `type`, `title`, `name`, `uri`.
- `activate` возвращает факт активации и `uri` активированного объекта.
- `close` возвращает факт закрытия и `uri` закрытого окна/формы.
- `open_form` возвращает факт открытия, `target` и URI открытой формы, если форма однозначно определена.

## Обработка ошибок
- `-32602` при некорректном URI или неподдерживаемом типе URI.
- `UI_NOT_FOUND` при отсутствии объекта по URI.
- `UI_AMBIGUOUS_OBJECT` при неоднозначном разрешении формы или control.
- `-32603` при ошибке вызова методов тест-клиента.

## Проверки
- `find` находит окна, формы и control и возвращает их URI.
- `activate` работает для окна, формы и control.
- `close` работает только для окна и формы.
- `open_form({target})` вызывает `ExecuteCommand` активного окна.

## Критерии готовности
- Все четыре инструмента видны в публичном контракте.
- URI объектов пригодны для цепочки `find -> activate/close`.
- `open_form` не принимает дополнительные аргументы, кроме `target`.

## Риски и неизвестные
- Нативный `FindObjects` может по-разному поддерживать фильтр по заголовку на разных типах корня поиска.
- Форму после `open_form` не всегда получится однозначно определить только по факту открытия.
