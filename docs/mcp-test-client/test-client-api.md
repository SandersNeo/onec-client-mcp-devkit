# API Test Client (MCP)

## Объектная модель тест‑клиента 1С

### `ТестируемоеПриложение`

Корневой объект. Создаётся через `Новый ТестируемоеПриложение("127.0.0.1", Порт)` или без параметров.

Методы:
- `УстановитьСоединение()` — устанавливает соединение с тестируемым приложением.
- `РазорватьСоединение()` — разрывает соединение.
- `ПолучитьАктивноеОкно()` → `ТестируемоеОкноКлиентскогоПриложения` — возвращает активное окно.
- `НайтиОбъект()` → `ТестируемоеОкноКлиентскогоПриложения | ТестируемаяФорма | ... | Неопределено` — поиск одного объекта по иерархии.
- `НайтиОбъекты()` → `ФиксированныйМассив` — поиск всех подходящих объектов по иерархии.
- `ПолучитьОбъект()` → аналогично `НайтиОбъект`, но с ожиданием.
- `ПолучитьПодчиненныеОбъекты()` → `ФиксированныйМассив` — объекты верхнего уровня (окна).
- `ОжидатьОтображениеОбъекта()` → `Булево` — ожидает появления объекта.
- `ОжидатьСостояния()` → `Булево` — ожидает наступления состояния (проверка раз в секунду).
- `ПолучитьТекущуюИнформациюОбОшибке()` → `ИнформацияОбОшибке | Неопределено`.
- `УстановитьМаксимальноеВремяВыполненияДействия()` / `ПолучитьМаксимальноеВремяВыполненияДействия()`.
- `ПолучитьНакопленныеПоказателиПроизводительности()` → `Структура` / `ОчиститьНакопленныеПоказателиПроизводительности()`.
- `УстановитьРезультатДиалогаВыбораФайла()` / `ОчиститьРезультатДиалогаВыбораФайла()`.
- `НачатьЗаписьЖурналаДействийПользователя()` / `ПриостановитьЗаписьЖурналаДействийПользователя()` / `ПродолжитьЗаписьЖурналаДействийПользователя()` / `ЗавершитьЗаписьЖурналаДействийПользователя()` → `Строка` / `ПрерватьЗаписьЖурналаДействийПользователя()`.

### `ТестируемоеОкноКлиентскогоПриложения`

Свойства: `Заголовок` (Строка), `НавигационнаяСсылка` (Строка), `Основное` (Булево), `НачальнаяСтраница` (Булево).

Методы:
- `Активизировать()` — активизирует окно.
- `Закрыть()` — закрывает окно.
- `ВыполнитьКоманду()` — выполняет команду (используется для открытия форм по навигационной ссылке).
- `НайтиОбъект()` / `НайтиОбъекты()` / `ПолучитьОбъект()` / `ПолучитьПодчиненныеОбъекты()` — поиск по иерархии.
- `ПолучитьКомандныйИнтерфейс()` → `ТестируемыйКомандныйИнтерфейсОкна`.
- `ПолучитьТекстыСообщенийПользователю()` → `ФиксированныйМассив`.
- `ВыбратьСообщениеПользователю()` — имитирует щелчок на сообщении.
- `ЗакрытьПанельСообщенийПользователю()`.
- `ПерейтиКНачальнойСтранице()` / `ПерейтиКПредыдущемуОкну()` / `ПерейтиКСледующемуОкну()` — только для основного окна.

### `ТестируемаяФорма`

Свойства: `ИмяФормы` (Строка), `ТекстЗаголовка` (Строка).

Методы:
- `Активизировать()`.
- `НайтиОбъект()` / `НайтиОбъекты()` / `ПолучитьОбъект()` / `ПолучитьПодчиненныеОбъекты()` / `ПолучитьРодителя()`.
- `ОжидатьЗакрытие()` → `Булево`.
- `ВыполнитьВыборИзМеню()` — выбор из открытого меню по представлению.
- `ВыполнитьВыборИзСписка()` — выбор из открытого списка выбора.
- `НайтиКнопкуПоУмолчанию()` → `ТестируемаяКнопкаФормы | Неопределено`.
- `ПолучитьКоманднуюПанель()` → `ТестируемаяГруппаФормы`.
- `ПолучитьПредставлениеСпискаВыбора()` → `ФиксированныйМассив` — элементы списка выбора.
- `ПолучитьТекущийЭлемент()` → `ТестируемоеПолеФормы | ТестируемаяГруппаФормы | ТестируемаяКнопкаФормы | ТестируемаяТаблицаФормы | ТестируемаяДекорацияФормы`.
- `ТекущаяДоступность()` → `Булево`.
- `ТекущаяМодифицированность()` → `Булево`.
- `ТекущееТолькоПросмотр()` → `Булево`.
- `ПерейтиКПредыдущемуЭлементу()` / `ПерейтиКСледующемуЭлементу()`.

## Tools

### `test_client_start`
Запускает процесс тест‑клиента 1С и подключается к нему.
Строка запуска формируется автоматически: `КаталогПрограммы()` + `СтрокаСоединенияИнформационнойБазы()` + ключи клиента.

Параметры:
- `port` — `TPort` для подключения (опционально, по умолчанию `1538`).

Результат:
- `started` — булево.
- `connected` — булево.
- `details` — строка или структура.

### `test_client_stop`
Разрывает соединение и пытается закрыть тест‑клиент.

Параметры:
- отсутствуют.

Результат:
- `stopped` — булево.

### `find`
Ищет окна, формы и элементы интерфейса тест‑клиента.

Параметры:
- `root_uri` — URI корневого объекта поиска (опционально, по умолчанию поиск от `ТестируемоеПриложение`).
- `type` — `window|form|group|button|table|field|decoration` (опционально).
- `title` — маска заголовка с поддержкой `*` и `?` (опционально).
- `name` — маска имени объекта с поддержкой `*` и `?` (опционально).
- `timeout_sec` — таймаут поиска в секундах (опционально).

Результат:
- массив объектов `type`, `title`, `name`, `uri`.

Ошибки:
- `-32602` — некорректные параметры или неподдерживаемый URI.
- `UI_NOT_READY` — тест‑клиент не подключен.
- `UI_NOT_FOUND` — объект не найден.
- `UI_AMBIGUOUS_OBJECT` — URI разрешается неоднозначно.
- `-32603` — ошибка вызова методов тест‑клиента.

### `activate`
Активизирует окно, форму или control по URI.

Параметры:
- `uri` — `window://...`, `form://...` или `control://...`.

Результат:
- объект `type`, `title`, `name`, `uri`.

Ошибки:
- `-32602`, `UI_NOT_READY`, `UI_NOT_FOUND`, `UI_AMBIGUOUS_OBJECT`, `-32603`.

### `click`
Нажимает на поддержанный элемент интерфейса по URI.

Параметры:
- `uri` — URI элемента (`control://...` для кнопки формы).

Результат:
- объект `type`, `title`, `name`, `uri`.
- `window_uri` — URI активного окна после действия.
- `form_uri` — URI активной формы после действия.
- `modal_opened` — булево, открылось ли новое модальное окно.

Ошибки:
- `ACTION_DENIED` — URI указывает на неподдерживаемый или недоступный для нажатия объект.
- `-32602`, `UI_NOT_READY`, `UI_NOT_FOUND`, `UI_AMBIGUOUS_OBJECT`, `-32603`.

### `input`
Вводит текст в поле формы по URI.

Параметры:
- `uri` — URI поля формы (`control://...`).
- `text` — строка для ввода.

Результат:
- объект `type`, `title`, `name`, `uri`.
- `value` — итоговое значение поля после ввода.
- `window_uri`, `form_uri`, `modal_opened`.

Ошибки:
- `ACTION_DENIED` — URI указывает на неподдерживаемое, выключенное или read-only поле.
- `-32602`, `UI_NOT_READY`, `UI_NOT_FOUND`, `UI_AMBIGUOUS_OBJECT`, `-32603`.

### `select`
Выполняет выбор значения в поле формы по URI.

Параметры:
- `uri` — URI поля формы (`control://...`).
- `value` — строковое значение выбора.
- `index` — индекс значения выбора.
- Должен быть передан ровно один из `value` или `index`.

Результат:
- объект `type`, `title`, `name`, `uri`.
- `value` — итоговое значение поля после выбора.
- `index` — выбранный индекс, если выбор выполнялся по индексу.
- `window_uri`, `form_uri`, `modal_opened`.

Ошибки:
- `ACTION_DENIED` — URI указывает на неподдерживаемое, выключенное или read-only поле.
- `-32602`, `UI_NOT_READY`, `UI_NOT_FOUND`, `UI_AMBIGUOUS_OBJECT`, `-32603`.

### `close`
Закрывает окно или форму по URI.

Параметры:
- `uri` — `window://...` или `form://...`.

Результат:
- объект `type`, `title`, `name`, `uri` закрытого окна или формы.

Ошибки:
- `ACTION_DENIED` — попытка закрыть `control://...`.
- `-32602`, `UI_NOT_READY`, `UI_NOT_FOUND`, `UI_AMBIGUOUS_OBJECT`, `-32603`.

### `open_form`
Открывает форму через `ExecuteCommand` активного окна.

Параметры:
- `target` — строка команды; на текущем этапе это навигационная ссылка.

Результат:
- `target` — исходная строка команды.
- `window_uri` — URI активного окна после выполнения команды.
- `form_uri` — URI открытой формы, если новая форма определена однозначно.

Ошибки:
- `-32602`, `UI_NOT_READY`, `-32603`.

## Resources
- `window://active` — активное окно.
- `window://{path}` — конкретное окно по навигационной ссылке.
- `form://{name}` — форма по имени формы; `resources/read` возвращает JSON-снимок формы с полями `type`, `uri`, `title`, `name`, `children`.
- `control://{form}/{name}` — элемент формы по имени формы и имени элемента; URI формируется только для именованных элементов.
- Шаблоны ресурсов публикуются через `resources/templates/list`.
- Поддерживаемый subset RFC 6570 ограничен выражениями `{var}`.
- `ui://active/form` и `ui://forms/{formId}` не публикуются.
