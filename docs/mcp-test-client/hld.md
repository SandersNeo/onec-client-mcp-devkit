# HLD: Test Client MCP

## Контекст
Решение добавляет слой управления тест‑клиентом 1С через MCP. Клиент MCP общается с сервером MCP внутри 1С через WebTransport‑компоненту.

## Компоненты
- MCP Client: агент, вызывающий tools и читающий resources.
- WebTransport AddIn: транспорт MCP (внешняя компонента).
- MCP Server (1C Client Extension): маршрутизация JSON‑RPC, реестры tools/resources/prompts.
- Test Client Control: адаптер к объектной модели тестирования 1С.
- UI Snapshot Builder: сбор дерева UI и свойств элементов.
- Notification Dispatcher: отправка уведомлений об изменениях.
- State Store: единая структура состояния MCP.

## Поток действий (tools)
- Клиент вызывает `tools/call`.
- Сервер валидирует параметры, проверяет необходимость подтверждения.
- Адаптер выполняет действие в тест‑клиенте.
- Сервер возвращает результат, пишет лог.

## Поток чтения (resources)
- Клиент вызывает `resources/read` по URI.
- Сервер формирует снимок UI.
- Сервер возвращает результат в формате ресурса.

## Уведомления
- Сервер отправляет уведомления клиенту при изменениях UI.
- Сервер отправляет `listChanged` для tools/resources/prompts при изменении реестров.

## Стратегия безопасности
- Разделение действий на безопасные и опасные.
- Требование подтверждения для опасных действий.
- Аудит всех пользовательских действий.

## Неизвестно (заглушка)
- Точный механизм подписки клиента на UI‑события.
- Формат и частота уведомлений при высокой частоте изменений UI.
