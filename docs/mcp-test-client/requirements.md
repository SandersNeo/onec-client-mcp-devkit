# Требования к решению Test Client MCP

## Цели
- Обеспечить агентам безопасное выполнение пользовательских действий в 1С через MCP.
- Обеспечить исследование интерфейса 1С через MCP (снимки форм и элементов управления).
- Поддержать уведомления об изменениях UI и реестров MCP.

## Область
- Внутри клиента 1С:Предприятие (расширение `ClientMcp`).
- Протокол MCP поверх существующего транспорта `AddIn.WebTransport.mcp`.
- Управление тест‑клиентом 1С через объектную модель тестирования.

## Функциональные требования
- Сервер MCP предоставляет `tools`, `resources`, `prompts`.
- Инструменты выполняют действия с интерфейсом тест‑клиента.
- Ресурсы дают дерево и свойства элементов UI.
- Промпты описывают высокоуровневые сценарии (создать, провести, закрыть и т.п.).
- Уведомления отправляются при изменении структуры UI и реестров.
- Состояние работы хранится в памяти MCP как единая структура.

## Требования к безопасности и подтверждениям
- «Опасные» действия требуют подтверждения пользователя.
- Список опасных действий фиксируется в реестре правил.
- Логирование всех действий и подтверждений обязательно.

## Нефункциональные требования
- Предсказуемые таймауты для действий UI и чтения ресурсов.
- Идемпотентность чтения ресурсов.
- Повторяемость результатов для снимков форм на одном состоянии UI.
- Минимизация побочных эффектов при исследовании UI.

## Ограничения
- Транспорт MCP остаётся WebTransport (внешняя компонента).
- Нет требований к серверному контуру 1С (только клиентский).

## Неизвестно (заглушка)
- Полный перечень «опасных» действий.
- Политика ретраев для команд UI.
- Требования к производительности на больших формах.
