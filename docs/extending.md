# Подключение собственных компонентов MCP

Для нового кода используйте public API v2:

- `Мсп_Сервер`
- `Мсп_Схема`
- `Мсп_Результаты`
- `Мсп_Ошибки`

Внутренние модули ядра остаются вне публичного контракта. Этот rollout не включает broad rename внутренних `*Клиент`-модулей: они по-прежнему считаются внутренней реализацией и не должны вызываться из прикладного кода напрямую.

Есть два способа добавить свои инструменты, ресурсы и промпты:

- провайдер в подсистеме `Мсп_Провайдеры`;
- внешняя обработка с прямой регистрацией через `Мсп_Сервер`.

## Запуск сервера

Для запуска сервера используйте структурный конфиг public API v2:

```bsl
&НаКлиенте
Асинх Процедура ЗапуститьMCP()
	ИнформацияОСервере = Мсп_Сервер.НоваяИнформацияОСервере(
		"my-mcp-server",
		"1.0.0",
		"My MCP Server",
		"Пример расширения MCP");

	ПараметрыСервера = Мсп_Сервер.НовыйПараметрыСервера(
		8080,
		"*",
		30,
		ИнформацияОСервере);

	Ждать Мсп_Сервер.Запустить(ПараметрыСервера);
КонецПроцедуры
```

## Способ 1: Провайдер

Провайдером является клиентский общий модуль, включенный в подсистему `Мсп_Провайдеры`.

При autodiscovery runtime:

1. пытается вызвать `Зарегистрировать(ИсточникРегистрации)`;
2. если метод отсутствует, использует compatibility fallback на `ЗарегистрироватьИнструменты`, `ЗарегистрироватьРесурсы`, `ЗарегистрироватьПромпты`;
3. сам передает строковый `ИсточникРегистрации`.

Для нового кода рекомендуется реализовывать именно `Зарегистрировать(ИсточникРегистрации)`.

### Минимальный каркас провайдера

```bsl
#Область ПрограммныйИнтерфейс

Процедура Зарегистрировать(ИсточникРегистрации) Экспорт
	ЗарегистрироватьИнструменты(ИсточникРегистрации);
	ЗарегистрироватьРесурсы(ИсточникРегистрации);
	ЗарегистрироватьПромпты(ИсточникРегистрации);
КонецПроцедуры

Процедура ЗарегистрироватьИнструменты(ИсточникРегистрации)
	Свойства = Новый Структура;
	Свойства.Вставить("query", Мсп_Схема.НовыйСвойствоСхемы("string", "Поисковый запрос"));
	Схема = Мсп_Схема.НовыйСхемаОбъекта(Свойства, ОбязательныеПоля());

	Инструмент = Мсп_Сервер.НовыйИнструмент(
		"my_search",
		"Выполняет поиск по базе данных",
		Схема,
		Новый ОписаниеОповещения("ОбработатьПоиск", МойПровайдерКлиент),
		"sync");

	Мсп_Сервер.ЗарегистрироватьИнструмент(Инструмент, ИсточникРегистрации);
КонецПроцедуры

Процедура ЗарегистрироватьРесурсы(ИсточникРегистрации)
	Ресурс = Мсп_Сервер.НовыйРесурс(
		"config://app",
		"Конфигурация приложения",
		"Текущие настройки приложения",
		"application/json",
		Новый ОписаниеОповещения("ОбработатьРесурсКонфига", МойПровайдерКлиент));

	Мсп_Сервер.ЗарегистрироватьРесурс(Ресурс, ИсточникРегистрации);

	Шаблон = Мсп_Сервер.НовыйШаблонРесурса(
		"document://{id}",
		"Документ",
		"Документ по идентификатору",
		"application/json",
		Новый ОписаниеОповещения("ОбработатьРесурсДокумента", МойПровайдерКлиент));

	Мсп_Сервер.ЗарегистрироватьШаблонРесурса(Шаблон, ИсточникРегистрации);
КонецПроцедуры

Процедура ЗарегистрироватьПромпты(ИсточникРегистрации)
	Аргументы = Новый Массив;
	Аргументы.Добавить(Новый Структура("name,description,required",
		"topic",
		"Тема для анализа",
		Истина));

	Промпт = Мсп_Сервер.НовыйПромпт(
		"analyze_topic",
		"Генерирует запрос для анализа темы",
		Аргументы,
		Новый ОписаниеОповещения("ОбработатьПромпт", МойПровайдерКлиент));

	Мсп_Сервер.ЗарегистрироватьПромпт(Промпт, ИсточникРегистрации);
КонецПроцедуры

Функция ОбязательныеПоля()
	Поля = Новый Массив;
	Поля.Добавить("query");
	Возврат Поля;
КонецФункции

#КонецОбласти
```

### Обработчик инструмента

Обработчик получает `Запрос` со следующими полями:

- `Аргументы` - аргументы вызова;
- `Контекст` - `kind`, `name` или `uri`, `arguments`, `sessionId`, `requestId`, `meta`, `progressToken`;
- `ИдентификаторЗадачи` - заполнен только для task-based вызова.

Для нового кода возвращайте явный результат через `Мсп_Результаты` и поднимайте ошибки через `Мсп_Ошибки`.

```bsl
Функция ОбработатьПоиск(Запрос, ДополнительныеПараметры) Экспорт
	СтрокаПоиска = Запрос.Аргументы.query;

	Если Не ЗначениеЗаполнено(СтрокаПоиска) Тогда
		Мсп_Ошибки.НекорректныеПараметры("query is required");
	КонецЕсли;

	Возврат Мсп_Результаты.Текст("Результат поиска: " + СтрокаПоиска);
КонецФункции
```

### Task-based инструмент

В public API v2 режим выполнения задается через `РежимВыполнения = "task"`.

```bsl
Процедура ЗарегистрироватьИнструменты(ИсточникРегистрации)
	Свойства = Новый Структура;
	Свойства.Вставить("ticks", Мсп_Схема.НовыйСвойствоСхемы("number", "Количество шагов"));
	Схема = Мсп_Схема.НовыйСхемаОбъекта(Свойства);

	Инструмент = Мсп_Сервер.НовыйИнструмент(
		"long_timer",
		"Долгая операция",
		Схема,
		Новый ОписаниеОповещения("ЗапуститьТаймер", МойПровайдерКлиент),
		"task");

	Мсп_Сервер.ЗарегистрироватьИнструмент(Инструмент, ИсточникРегистрации);
КонецПроцедуры

Функция ЗапуститьТаймер(Запрос, ДополнительныеПараметры) Экспорт
	ИдентификаторЗадачи = Запрос.ИдентификаторЗадачи;
	Если Не ЗначениеЗаполнено(ИдентификаторЗадачи) Тогда
		Мсп_Ошибки.СбойВыполнения("task-based invocation is required");
	КонецЕсли;

	Мсп_Сервер.УстановитьСтатусЗадачи(ИдентификаторЗадачи, "working", "Timer started");
	Мсп_Сервер.УведомитьОПрогрессеЗадачи(ИдентификаторЗадачи, 0, Запрос.Аргументы.ticks, "Task accepted");

	// Запланируйте выполнение любым подходящим способом
	// и по завершении явно завершите задачу:
	Мсп_Сервер.ЗавершитьЗадачуУспешно(
		ИдентификаторЗадачи,
		Мсп_Результаты.Текст("Timer completed"));
КонецФункции
```

Если задача завершается ошибкой, используйте `Мсп_Сервер.ЗавершитьЗадачуСОшибкой(...)`.

### Обработчик ресурса

```bsl
Функция ОбработатьРесурсКонфига(Запрос, ДополнительныеПараметры) Экспорт
	URI = Запрос.Контекст.uri;
	Данные = Новый Структура("версия,режим", "1.0", "production");
	Возврат Мсп_Результаты.Ресурс(URI, Данные);
КонецФункции

Функция ОбработатьРесурсДокумента(Запрос, ДополнительныеПараметры) Экспорт
	Аргументы = Запрос.Контекст.arguments;
	Ид = Мсп_Коллекции.Свойство(Аргументы, "id", "");

	Если Не ЗначениеЗаполнено(Ид) Тогда
		Мсп_Ошибки.НекорректныеПараметры("id is required");
	КонецЕсли;

	Данные = Новый Структура("id,title", Ид, "Документ " + Ид);
	Возврат Мсп_Результаты.Ресурс(Запрос.Контекст.uri, Данные);
КонецФункции
```

## Способ 2: Внешняя обработка

Если компонент нельзя встроить в расширение, регистрируйте его напрямую через `Мсп_Сервер` и передавайте строковый `ИсточникРегистрации`.

```bsl
&НаКлиенте
Процедура ПриОткрытии(Отказ)
	ЗарегистрироватьКомпоненты();
КонецПроцедуры

&НаКлиенте
Процедура ЗарегистрироватьКомпоненты()
	ИсточникРегистрации = "МояВнешняяОбработка";

	Свойства = Новый Структура;
	Свойства.Вставить("text", Мсп_Схема.НовыйСвойствоСхемы("string", "Текст для обработки"));
	Поля = Новый Массив;
	Поля.Добавить("text");
	Схема = Мсп_Схема.НовыйСхемаОбъекта(Свойства, Поля);

	Инструмент = Мсп_Сервер.НовыйИнструмент(
		"process_text",
		"Обрабатывает текст",
		Схема,
		Новый ОписаниеОповещения("ОбработатьТекст", ЭтотОбъект),
		"sync");

	Мсп_Сервер.ЗарегистрироватьИнструмент(Инструмент, ИсточникРегистрации);
КонецПроцедуры

&НаКлиенте
Функция ОбработатьТекст(Запрос, ДополнительныеПараметры) Экспорт
	Возврат Мсп_Результаты.Текст("Обработано: " + Запрос.Аргументы.text);
КонецФункции
```

## Результаты и ошибки

Для нового кода используйте только явные builders:

### Результаты

```bsl

Мсп_Результаты.JSON(Данные)
Мсп_Результаты.Ресурс(URI, Данные, MimeType)
Мсп_Результаты.Промпт(Сообщения)
```

### Ошибки

```bsl
Мсп_Ошибки.НекорректныеПараметры("...")
```

## Compatibility notes

- `НовыйОписаниеИнструмента`, `НовыйОписаниеРесурса`, `НовыйОписаниеШаблонаРесурса`, `НовыйОписаниеПромпта` сохранены только как compatibility aliases.
- `Асинхронный` остается compatibility-термином для legacy descriptor API. Для нового кода используйте `РежимВыполнения = "sync"` или `"task"`.
- `ЗарегистрироватьИнструменты`, `ЗарегистрироватьРесурсы`, `ЗарегистрироватьПромпты` у провайдера остаются compatibility path для legacy autodiscovery.
- `ОтменитьРегистрациюПровайдера` остается compatibility alias. Для нового кода используйте `Мсп_Сервер.ОтменитьРегистрацию(ИсточникРегистрации)`.
- Внутренние `*Клиент`-модули не переименовывались в рамках этого rollout и не являются частью публичного API.
