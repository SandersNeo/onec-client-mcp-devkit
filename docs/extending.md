# Подключение собственных компонентов MCP

Есть два способа добавить свои инструменты, ресурсы и промпты в MCP-сервер:

- **Провайдер** — общий модуль в подсистеме `Мсп_Провайдеры` (рекомендуется для расширений).
- **Внешняя обработка** — регистрация компонентов вручную из кода обработки.

---

## Способ 1: Провайдер (общий модуль)

Провайдер — это клиентский общий модуль, включённый в подсистему `Мсп_Провайдеры`. При запуске сервера система автоматически обнаруживает все такие модули и вызывает у них методы регистрации.

### Шаги

**1. Создайте клиентский общий модуль** (например, `МойПровайдерКлиент`).

**2. Включите модуль в подсистему `Мсп_Провайдеры`.**

**3. Реализуйте нужные методы** — только те, которые необходимы. Отсутствующие методы игнорируются.

```bsl
// МойПровайдерКлиент

#Область ПрограммныйИнтерфейс

Процедура ЗарегистрироватьИнструменты() Экспорт
    // регистрация инструментов
КонецПроцедуры

Процедура ЗарегистрироватьРесурсы() Экспорт
    // регистрация ресурсов
КонецПроцедуры

Процедура ЗарегистрироватьПромпты() Экспорт
    // регистрация промптов
КонецПроцедуры

#КонецОбласти
```

### Регистрация инструмента

```bsl
Процедура ЗарегистрироватьИнструменты() Экспорт
    Свойства = Новый Структура;
    Свойства.Вставить("query", Мсп_СхемаПараметров.НовыйСвойствоСхемы("string", "Поисковый запрос"));
    Схема = Мсп_СхемаПараметров.НовыйСхемаОбъекта(Свойства, ОбязательныеПоля());

    Мсп_Сервер.ЗарегистрироватьИнструмент(Мсп_Сервер.НовыйОписаниеИнструмента(
        "my_search",
        "Выполняет поиск по базе данных",
        Схема,
        Новый ОписаниеОповещения("ОбработатьПоиск", МойПровайдерКлиент),
        "МойПровайдер"));
КонецПроцедуры

Функция ОбязательныеПоля()
    Поля = Новый Массив;
    Поля.Добавить("query");
    Возврат Поля;
КонецФункции

// Параметры:
//  Запрос - Структура:
//   * Аргументы - Структура - аргументы вызова (Запрос.Аргументы.query и т.д.)
//   * Контекст - Структура - контекст вызова (sessionId, requestId и др.)
//  ДополнительныеПараметры - Произвольный
//
// Возвращаемое значение: любое значение — строка, структура, массив.
// Фреймворк автоматически обернёт результат в { content: [...] }.
Функция ОбработатьПоиск(Запрос, ДополнительныеПараметры) Экспорт
    Запрос = Запрос.Аргументы.query;
    // ... логика поиска ...
    Возврат "Результат поиска: " + Запрос;
КонецФункции
```

### Асинхронный инструмент

Для долгой операции зарегистрируйте инструмент с флагом `Асинхронный = Истина`.
В таком режиме MCP вызывает инструмент как task-based `tools/call`, а в `Запрос` передаётся настроенный `ОбработчикЗавершения`.

```bsl
Процедура ЗарегистрироватьИнструменты() Экспорт
    Свойства = Новый Структура;
    Свойства.Вставить("ticks", Мсп_СхемаПараметров.НовыйСвойствоСхемы("number", "Количество шагов"));
    Схема = Мсп_СхемаПараметров.НовыйСхемаОбъекта(Свойства);

    Мсп_Сервер.ЗарегистрироватьИнструмент(Мсп_Сервер.НовыйОписаниеИнструмента(
        "long_timer",
        "Долгая асинхронная операция",
        Схема,
        Новый ОписаниеОповещения("ЗапуститьТаймер", МойПровайдерКлиент),
        "МойПровайдер",
        Истина));
КонецПроцедуры

Функция ЗапуститьТаймер(Запрос, ДополнительныеПараметры) Экспорт
    ОбработчикЗавершения = Запрос.ОбработчикЗавершения;
    ИдентификаторЗадачи = Запрос.Контекст.taskId;

    Мсп_Сервер.УстановитьСтатусЗадачи(ИдентификаторЗадачи, "working", "Timer started");
    Мсп_Сервер.УведомитьОПрогрессеЗадачи(ИдентификаторЗадачи, 0, Запрос.Аргументы.ticks, "Task accepted");

    // Запланируйте выполнение любым своим способом
    // и по завершении вызовите обработчик:
    ВыполнитьОбработкуОповещения(ОбработчикЗавершения, "Timer completed");
КонецФункции
```

Если операция завершилась ошибкой, передайте в обработчик результат
`Мсп_Сервер.ОшибкаИнструмента("ACTION_FAILED", "описание ошибки")`.
Для отмены провайдер может проверять `Мсп_Сервер.ЗадачаОтменена(taskId)` и самостоятельно завершать локальное выполнение.

### Регистрация ресурса

```bsl
Процедура ЗарегистрироватьРесурсы() Экспорт
    // Статический ресурс
    Мсп_Сервер.ЗарегистрироватьРесурс(Мсп_Сервер.НовыйОписаниеРесурса(
        "config://app",
        "Конфигурация приложения",
        "Текущие настройки приложения",
        Новый ОписаниеОповещения("ОбработатьРесурсКонфига", МойПровайдерКлиент),
        "МойПровайдер"));

    // Шаблонный ресурс (URI с параметрами)
    Мсп_Сервер.ЗарегистрироватьШаблонРесурса(Мсп_Сервер.НовыйОписаниеШаблонаРесурса(
        "document://{id}",
        "Документ",
        "Документ по идентификатору",
        Новый ОписаниеОповещения("ОбработатьРесурсДокумента", МойПровайдерКлиент),
        "МойПровайдер"));
КонецПроцедуры

// Обработчик статического ресурса.
// Возвращаемое значение: структура/массив (сериализуется в JSON),
// строка (передаётся как text/plain),
// или Мсп_РесурсыКлиент.ОшибкаРесурса(...) при ошибке.
Функция ОбработатьРесурсКонфига(Запрос, ДополнительныеПараметры) Экспорт
    URI = Запрос.Контекст.uri;
    Данные = Новый Структура("версия, режим", "1.0", "production");
    Возврат Мсп_РесурсыКлиент.РезультатРесурса(URI, Данные);
КонецФункции

// Обработчик шаблонного ресурса.
// Аргументы шаблона доступны через Запрос.Контекст.arguments.
Функция ОбработатьРесурсДокумента(Запрос, ДополнительныеПараметры) Экспорт
    URI = Запрос.Контекст.uri;
    Аргументы = Запрос.Контекст.arguments;
    Ид = Мсп_Коллекции.Свойство(Аргументы, "id", "");

    Если Не ЗначениеЗаполнено(Ид) Тогда
        Возврат Мсп_РесурсыКлиент.ОшибкаРесурса("INVALID_PARAMS", "id is required");
    КонецЕсли;

    Данные = Новый Структура("id, title", Ид, "Документ " + Ид);
    Возврат Мсп_РесурсыКлиент.РезультатРесурса(URI, Данные);
КонецФункции
```

### Регистрация промпта

```bsl
Процедура ЗарегистрироватьПромпты() Экспорт
    Аргументы = Новый Массив;
    Арг = Новый Структура("name, description, required", "topic", "Тема для анализа", Истина);
    Аргументы.Добавить(Арг);

    ОписаниеПромпта = Мсп_РеестрКлиент.НовыйОписаниеПромпта();
    ОписаниеПромпта.Имя = "analyze_topic";
    ОписаниеПромпта.Описание = "Генерирует запрос для анализа темы";
    ОписаниеПромпта.СхемаПараметров = Новый Структура("arguments", Аргументы);
    ОписаниеПромпта.Обработчик = Новый ОписаниеОповещения("ОбработатьПромпт", МойПровайдерКлиент);
    ОписаниеПромпта.Провайдер = "МойПровайдер";
    Мсп_РеестрКлиент.ЗарегистрироватьПромпт(ОписаниеПромпта);
КонецПроцедуры

// Возвращаемое значение: строка, массив сообщений,
// или структура { messages: [...] }.
Функция ОбработатьПромпт(Запрос, ДополнительныеПараметры) Экспорт
    Тема = Мсп_Коллекции.Свойство(Запрос.Аргументы, "topic", "");
    Возврат "Проанализируй тему: " + Тема;
КонецФункции
```

## Способ 2: Внешняя обработка

Если нет возможности добавить общий модуль в расширение (например, компоненты поставляются отдельно), можно зарегистрировать их напрямую из кода внешней обработки.

### Шаги

**1. Создайте внешнюю обработку** с клиентским модулем.

**2. Вызовите методы `Мсп_Сервер` напрямую** — в любой момент после инициализации сервера.

```bsl
// Модуль формы или клиентский модуль внешней обработки

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    ЗарегистрироватьКомпоненты();
КонецПроцедуры

&НаКлиенте
Процедура ЗарегистрироватьКомпоненты()
    ЗарегистрироватьИнструменты();
    ЗарегистрироватьРесурсы();
КонецПроцедуры

&НаКлиенте
Процедура ЗарегистрироватьИнструменты()
    Свойства = Новый Структура;
    Свойства.Вставить("text", Мсп_СхемаПараметров.НовыйСвойствоСхемы("string", "Текст для обработки"));
    Поля = Новый Массив;
    Поля.Добавить("text");
    Схема = Мсп_СхемаПараметров.НовыйСхемаОбъекта(Свойства, Поля);

    Мсп_Сервер.ЗарегистрироватьИнструмент(Мсп_Сервер.НовыйОписаниеИнструмента(
        "process_text",
        "Обрабатывает текст",
        Схема,
        Новый ОписаниеОповещения("ОбработатьТекст", ЭтотОбъект),
        "МояОбработка"));
КонецПроцедуры

&НаКлиенте
Процедура ЗарегистрироватьРесурсы()
    Мсп_Сервер.ЗарегистрироватьРесурс(Мсп_Сервер.НовыйОписаниеРесурса(
        "processing://status",
        "Статус обработки",
        "Текущий статус модуля обработки",
        Новый ОписаниеОповещения("ОбработатьРесурсСтатуса", ЭтотОбъект),
        "МояОбработка"));
КонецПроцедуры

&НаКлиенте
Функция ОбработатьТекст(Запрос, ДополнительныеПараметры) Экспорт
    Текст = Запрос.Аргументы.text;
    // ... логика обработки ...
    Возврат "Обработано: " + Текст;
КонецФункции

&НаКлиенте
Функция ОбработатьРесурсСтатуса(Запрос, ДополнительныеПараметры) Экспорт
    URI = Запрос.Контекст.uri;
    Данные = Новый Структура("активен", Истина);
    Возврат Мсп_РесурсыКлиент.РезультатРесурса(URI, Данные);
КонецФункции
```

### Отмена регистрации при закрытии

Если обработка закрывается, нужно снять регистрацию своих компонентов, чтобы не оставлять «мёртвые» обработчики в реестре:

```bsl
&НаКлиенте
Процедура ПередЗакрытием(Отказ, СтандартнаяОбработка)
    Мсп_РеестрКлиент.ОтменитьРегистрациюПровайдера("МояОбработка");
КонецПроцедуры
```

## Справочник: возвращаемые значения обработчиков

### Инструмент

Обработчик может вернуть любое из следующего — фреймворк нормализует автоматически:

| Тип возвращаемого значения | Результат |
|---|---|
| `Строка` | `{ content: [{ type: "text", text: "..." }] }` |
| `Массив` | `{ content: [...] }` |
| `Структура` с полем `content` | передаётся как есть |
| Любой другой тип | сериализуется в JSON, оборачивается в `text` |

### Ресурс

| Тип возвращаемого значения | Результат |
|---|---|
| `Мсп_РесурсыКлиент.РезультатРесурса(URI, Данные)` | `{ contents: [{ uri, text, mimeType }] }` |
| `Структура` с полем `contents` | передаётся как есть |
| `Строка` | `{ contents: [{ uri, text, mimeType }] }` |
| `Мсп_РесурсыКлиент.ОшибкаРесурса(Код, Сообщение)` | JSON-RPC ошибка |

Коды ошибок ресурса: `INVALID_URI`, `INVALID_PARAMS`, `ACTION_FAILED`, `ACTION_DENIED`.

### Промпт

| Тип возвращаемого значения | Результат |
|---|---|
| `Строка` | `{ messages: [{ role: "user", content: [{ type: "text", text: "..." }] }] }` |
| `Массив` | `{ messages: [...] }` |
| `Структура` с полем `messages` | передаётся как есть |

## Контекст вызова

Каждый обработчик получает `Запрос.Контекст` со следующими полями:

| Поле | Тип | Описание |
|---|---|---|
| `kind` | Строка | `"tools"`, `"resources"` или `"prompts"` |
| `name` / `uri` | Строка | Имя инструмента/промпта или URI ресурса |
| `arguments` | Структура | Аргументы вызова |
| `sessionId` | Строка | Идентификатор сессии MCP |
| `requestId` | Произвольный | Идентификатор запроса JSON-RPC |
| `raw` | Произвольный | Исходные данные запроса |
| `meta` | Структура | Метаданные запроса |

## Сравнение подходов

| | Провайдер | Внешняя обработка |
|---|---|---|
| Регистрация | Автоматически при старте сервера | Вручную из кода |
| Жизненный цикл | Весь сеанс | Пока открыта обработка |
| Отмена регистрации | Не требуется | Нужно вызвать `ОтменитьРегистрациюПровайдера` |
| Подходит для | Расширений конфигурации | Внешних поставляемых модулей |
